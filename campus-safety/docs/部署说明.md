# 黔视护苗 · 部署与迁移安装说明

本文说明如何在另一台电脑（以 Windows 笔记本为主）上从零安装并运行本系统。

---

## 一、项目结构与技术栈

根目录 **`campus-safety`** 为「校园安全智能监控系统」主工程：

| 部分 | 技术栈 | 默认端口 |
|------|--------|----------|
| 后端 | Python 3 + Flask + Flask-SocketIO（gevent）+ OpenCV + Ultralytics（YOLO）+ 阿里云 DashScope 等 | **5000**（监听 `0.0.0.0`） |
| 前端 | Vue 3 + Vite + Element Plus + Pinia + Socket.IO + ECharts + 高德地图（无人机规划页） | **3000** |

业务数据主要位于 **`backend/data/`**（JSON 配置与会话等）与 **`backend/static/`**（告警视频片段、持续录像等）。当前版本 **不依赖 MySQL** 等外部数据库。

---

## 二、迁移时需要拷贝的内容

1. **完整 `campus-safety` 目录**（或通过 Git 克隆同一代码）。
2. **不建议直接拷贝** 源电脑上的 **`venv/`** 与 **`frontend/node_modules/`**：换机器后应在目标机上 **新建虚拟环境** 并执行 **`npm install`**，避免路径与二进制不兼容。
3. **YOLO 权重文件**（`.pt`）：放到 **`backend/`** 目录下，或与环境变量 **`YOLO_MODEL_PATH`** 所填路径一致。若使用 `yolov8n.pt` 且本地无文件，Ultralytics 可能在首次运行时自动下载（需联网）。
4. **环境配置**：将 **`backend/.env.example`** 复制为 **`backend/.env`**，按下一节填写；**勿将含真实密钥的 `.env` 提交到版本库**。

---

## 三、目标机环境准备

| 依赖 | 说明 |
|------|------|
| Python | 建议 **3.10 或 3.11**（64 位），安装时勾选将 Python 加入 PATH |
| Node.js | 建议 **LTS（18.x / 20.x）**，自带 npm |
| ffmpeg | 用于持续录像与视频转码；下载后加入系统 PATH，或在 `.env` 中配置 **`FFMPEG_PATH`** 为 `ffmpeg.exe` 的绝对路径 |
| （可选）NVIDIA GPU | 安装与 CUDA 版本匹配的 PyTorch 可加速检测；仅用 CPU 也可运行 |

**说明：** `config.py` 中 `FFMPEG_PATH` 的默认值可能指向某台开发机的路径，**换电脑后务必在 `.env` 中覆盖为当前机器的路径**，否则录像相关功能易失败。

检测框中文标签在 Windows 下会尝试读取系统字体目录（如黑体、微软雅黑），一般无需额外安装字体。

## 四、后端安装与启动

本项目建议使用 **Conda** 创建虚拟环境（你也可以使用 `venv`，但本文以 Conda 为准）。

### 4.1 创建并激活 Conda 环境

在 **Anaconda Prompt / PowerShell / cmd** 中执行（首次部署需要创建一次）：

```powershell
conda create -n campus_env python=3.14.3 -y
conda activate campus_env
python -m pip install -U pip
```

> 提示：若你本机的 Conda 源暂时没有 `python=3.14.3`，请改成可用的 Python 版本（例如 3.10/3.11），其余步骤不变。

### 4.2 安装后端依赖

进入后端目录并安装依赖：

```powershell
cd "你的路径\campus-safety\backend"
pip install -r requirements.txt
```

配置环境变量文件：

```powershell
copy .env.example .env
```

编辑 **`backend/.env`**，至少关注以下项（与 `.env.example` 一致）：

- **`DASHSCOPE_API_KEY`**：阿里云百炼 / DashScope API Key；未配置或网络异常时，部分 AI 能力会进入 Mock 模式，仍可演示。
- **`YOLO_MODEL_PATH`**：模型文件名或路径，例如 `yolov8n.pt`、`best.pt`。
- **`FFMPEG_PATH`**（推荐显式配置）：本机 `ffmpeg.exe` 绝对路径。
- **`TTS_APP_KEY`**、**`TTS_TOKEN`**：语音播报相关（按需要填写）。
- **`SECRET_KEY`**：生产环境请改为随机长字符串。
- **`DEBUG`**：`true` / `false`。

### 4.3 启动后端

```powershell
conda activate campus_env
python app.py
```

服务成功时监听 **5000** 端口。

## 五、前端安装与启动

**新开一个终端**，进入前端目录：

```powershell
cd "你的路径\campus-safety\frontend"
npm install
npm run dev
```

浏览器访问：**http://localhost:3000**

`vite.config.js` 已将 **`/api`**、**`/static`**、**`/socket.io`** 代理到 **http://localhost:5000**，因此本机使用时须 **先启动后端，再启动前端**。

默认测试账号见同目录下的 **[用户手册.md](./用户手册.md)** 中「系统启动」一节。

---

## 六、一键启动（Windows · `start.bat`）

在 **已完成** 第四节、第五节中的环境准备（含已创建 Conda 环境 `campus_env`、已 `npm install`、已配置 `backend\.env` 等）后，可双击一键启动，无需每次手动开两个终端。

| 项 | 说明 |
|----|------|
| 文件位置 | **`campus-safety\start.bat`**（与 `backend`、`frontend` 同级） |
| 操作 | 双击 `start.bat` |
| 行为 | 依次在新窗口启动后端、前端；约等待数秒后 **自动打开浏览器** `http://localhost:3000` |
| 停止 | 关闭标题为 **「黔视护苗-后端」**、**「黔视护苗-前端」** 的两个命令行窗口即可 |
| 启动器窗口 | 双击后会出现短暂提示窗口，约 **5 秒** 后自动关闭；**勿** 用关闭该小窗口来停止服务 |

若未安装 Conda / 找不到 conda，或未安装前端依赖，脚本会提示错误并退出。

> **说明：** 脚本会优先在系统 PATH 中定位 `conda`，若找不到也会尝试常见安装目录（如 `C:\ProgramData\Anaconda3\condabin\conda.bat`、`%USERPROFILE%\Miniconda3\condabin\conda.bat` 等）。找到后会执行 `conda activate campus_env` 再启动后端。若你的环境名不是 `campus_env`，请编辑 `start.bat` 顶部的 `CONDA_ENV`。

---

## 七、启动顺序与简单自检

1. 启动 **后端**（端口 5000），再启动 **前端**（端口 3000）；或使用 **第六节** 的 `start.bat` 一并启动。
2. 浏览器打开 `http://localhost:3000`，登录后在 **系统设置** 中可查看 API 等连接状态。
3. **实时监控** 中可使用本机摄像头索引 `0`、上传本地视频文件或 RTSP 等（详见用户手册）。

---

## 八、局域网多设备访问说明（限制）

当前前端部分页面将视频流、静态资源地址、WebSocket 写为 **`http://localhost:5000`**（如监控大屏、事件录像、Socket 连接等）。

因此：

- **仅在本机浏览器访问 `http://localhost:3000`**：行为与开发配置一致。
- **在局域网另一台电脑/手机只访问 `http://笔记本IP:3000`**：页面可能能打开，但视频与实时连接仍指向「访问者自己机器」的 5000 端口，通常 **无法正常播放**。若需在局域网多终端演示，应将上述地址改为 **`http://<运行后端的机器局域网IP>:5000`** 或通过环境变量统一配置（需改前端代码）。

## 九、常见问题

| 现象 | 处理建议 |
|------|----------|
| `pip install` 慢或失败 | 使用国内 PyPI 镜像；若单独安装 PyTorch，请按 [PyTorch 官网](https://pytorch.org/) 选择 CPU/CUDA 版本后再装其余依赖。 |
| 模型无法加载 | 检查 `YOLO_MODEL_PATH` 与文件是否存在于目标机路径。 |
| 录像、转码报错 | 检查 `ffmpeg` 是否在 PATH 中，或 `.env` 中 `FFMPEG_PATH` 是否正确。 |
| 前端无数据或视频黑屏 | 确认后端已启动且 5000 未被占用；跨机访问见第八节。 |
| `start.bat` 提示找不到 venv 或 node_modules | 按第四、五节先完成安装；检查目录是否被移动、杀毒软件是否拦截 `.bat`。 |
