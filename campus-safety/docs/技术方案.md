## 二、核心技术模块

### 2.1 视频流处理

- 支持：RTSP实时流、本地视频文件（循环播放）、手动上传、本地摄像头（设备索引0/1/2...）
- OpenCV 多线程读帧，环形缓冲区保存最近5秒帧（触发前上下文），时间戳补偿保证播放速度与原视频一致
- MJPEG HTTP流推送前端，每路独立连接，约25fps，支持Range请求实现视频流式播放
- **检测框异步叠加**：独立检测线程缓存最新检测结果，推流线程直接取缓存画框，互不阻塞，保证视频流畅
- 触发后继续录制5秒，合成完整10秒事件片段，ffmpeg转H.264保证浏览器兼容
- 摄像头接入状态持久化，重启后自动恢复

### 2.2 YOLO 行为检测

| 参数 | 配置 |
|------|------|
| 模型 | YOLO26x（YOLO11系列，高精度，已微调） |
| 微调类别 | fighting（打架）/ falling（跌倒）/ intrusion（入侵）/ objects（正常） |
| 置信度阈值 | 0.5（系统设置可实时调整，立即生效） |
| 检测频率 | 每3帧检测一次（系统设置可实时调整） |
| 冷却机制 | 同摄像头同行为30秒内不重复触发（系统设置可实时调整） |
| 检测框显示 | PIL渲染中文标签，实时叠加在MJPEG流上 |
| 俯视角支持 | 训练数据包含无人机俯视角，适配无人机场景 |

**动态配置**：所有检测参数通过 `/api/detection/config` 接口实时更新，无需重启后端即可生效。

### 2.3 Qwen-VL 多帧视频分析

- 模型：`qwen-vl-max-latest`
- 触发时机：YOLO检测到危险行为后，从存档片段均匀抽取关键帧（每秒1帧，最多16帧）
- 作用：基于时序多帧理解事件经过，二次确认是否真实危险，降低误报率至5%以下
- 降级机制：API不可用时自动切换Mock数据，保证演示流畅

### 2.4 Qwen LLM 智能分析

- 模型：`qwen-max-latest`
- 功能一：生成结构化预警报告（事件概述、风险等级、处置建议）
- 功能二：动态生成广播播报文字（权威简洁，适合校园广播）
- 功能三：自然语言查询历史事件（对话式管理）
- 功能四：每日安全日报自动生成
- 功能五：AI风险预测与态势分析（结合历史数据预测高危时段和区域）
- 功能六：AI自动生成今日巡逻任务（基于风险分析结果）

### 2.5 校园安全指数计算

采用**加权风险评分模型**：

```
weights = {fighting:10, intrusion:10, bullying:8, falling:6, gathering:5, person:3}
risk_score = min(100, Σ(weights[行为类型] × 该类型事件数))
risk_level = "高" if score≥60 else "中" if score≥30 else "低"
```

不同危险行为赋予不同权重，累加后上限100，直观反映校园安全态势。

### 2.6 语音广播

- 优先方案：阿里云TTS（sambert系列），配置TTS_APP_KEY后启用
- 兜底方案：浏览器Web Speech API，无需任何配置，中文语音朗读广播内容
- 支持配置重复次数（1~10次）和各行为类型的自定义广播内容
- 两种方案自动切换，保证演示时广播功能始终可用

### 2.7 边缘计算方案（无人机）

```
无人机端（边缘节点，如NVIDIA Jetson Nano）
    ↓ 本地运行轻量化 YOLO
    ↓ 仅检测到异常时
    ↓ 上传关键帧 + 短视频片段（~1-5MB）
服务器端
    ↓ Qwen-VL 多帧二次确认
    ↓ 生成报告 + 触发预警
```

相比全量视频流传输，带宽需求降低约90%。

### 2.8 无人机智能路径规划

基于高德地图API（v1.4.15）实现：
1. 管理员在地图上标记学校位置、建筑物（含高度）、学生回家路线
2. 系统按配置间距（默认100米）沿路线均匀插值生成航点
3. 每个航点飞行高度 = `max(基础高度, 附近建筑高度 + 安全余量)`
4. 生成完整航点序列（起飞→巡逻→返航），计算预计飞行时间
5. 导出KML格式航线文件，可直接导入大疆DJI Pilot 2执行
6. 地图上显示路线两侧半透明缓冲区，直观展示巡逻覆盖范围

### 2.9 预警响应链路（全自动，<3秒）

```
YOLO检测触发（异步检测线程）
  → Qwen-VL 多帧确认（降低误报）
  → 保存事件前后视频片段（H.264，浏览器可播放）
  → Qwen 生成报告 + 广播文字
  → SQLite 持久化存储
  → 并行执行：
      ├── Web Speech API / 阿里云TTS 语音广播（可配置重复次数）
      ├── 模拟短信通知管理人员
      └── WebSocket 推送 → 前端弹出警报弹窗 + 警报音效
```

### 2.10 持续录像（NVR模式）

- 支持对指定摄像头在配置时间段内持续录制（00:00~23:59任意时间段）
- 按分段时长（默认30分钟）自动切割，ffmpeg转H.264
- 配置项：启用开关、录像时间段、分段时长、指定摄像头、留存天数
- 配置持久化到 `data/recording_config.json`，重启后自动恢复
- 每天自动清理超过留存天数的录像文件
- 历史录像支持按摄像头和日期筛选，在线播放（支持Range请求）或下载

### 2.11 数据统计与可视化

- 事件趋势折线图（近7/14/30天，多行为类型对比）
- 行为类型分布饼图
- 各区域事件分布柱状图
- 事件时段分布柱状图
- 校园风险热力图（基于摄像头位置和事件频次）
- 支持时间范围筛选，数据每15秒自动刷新

### 2.12 安全大屏

专为答辩和领导汇报设计的全屏可视化大屏（进入自动全屏）：
- 左栏：校园安全指数（加权评分）、高危区域TOP3、AI风险预测、高危时段分布
- 中栏：实时多路视频监控（1-4路自适应布局，带检测框）、今日事件概览
- 右栏：最新告警（按行为类型区分颜色）、AI防控建议、行为类型统计横向柱状图

### 2.13 多角色权限系统

| 角色 | 账号 | 权限范围 |
|------|------|---------|
| 校长（principal） | admin | 全部功能 |
| 安全主任（manager） | safety | 监控、事件、历史录像、统计、查询、巡逻、无人机规划、大屏 |
| 保安（guard） | guard | 实时监控、事件记录、历史录像、巡逻任务 |

JWT Token 认证，前端路由守卫，12小时有效期。

## 四、模型微调说明

### 数据集来源
整合5个开源数据集，共约XXXX张标注图片：
- Fall Detection Dataset（跌倒检测）
- Fighting Detection Dataset（打架检测）
- Fighting Recognition Dataset（打架识别）
- Intrusion Detection by YOLOv11（入侵检测）
- Violence Detection Dataset（暴力检测）

### 类别映射
| 最终类别 | 原始类别 | 含义 |
|---------|---------|------|
| fighting (ID=0) | Fighting / fight / violence | 打架/暴力 |
| falling (ID=1) | Fall | 人员跌倒 |
| intrusion (ID=2) | Intrusion detected | 陌生人入侵 |
| objects (ID=3) | Non-fall / nofight / objects | 正常（不触发预警） |

### 训练配置
- 基础模型：YOLO26x（YOLO11系列高精度版本）
- 训练轮数：100 epochs
- 输入尺寸：640×640
- 数据增强：Mosaic、Mixup、HSV增强、随机翻转

## 五、部署说明

### 环境要求
- Python 3.10+（推荐 Anaconda）
- Node.js 18+
- ffmpeg（视频转码，需配置路径）

### 后端启动
```bash
cd backend
pip install -r requirements.txt
# 复制 .env.example 为 .env 并填入 API Key
python app.py
```

### 前端启动
```bash
cd frontend
npm install
npm run dev
```

访问 http://localhost:3000，使用以下账号登录：

| 账号 | 密码 | 角色 |
|------|------|------|
| admin | admin123 | 校长（全部权限） |
| safety | safety123 | 安全主任 |
| guard | guard123 | 保安 |

### 关键配置（backend/.env）
```
DASHSCOPE_API_KEY=your-key    # 阿里云百炼 API Key
YOLO_MODEL_PATH=best.pt       # 微调后的模型文件
FFMPEG_PATH=C:\...\ffmpeg.exe # ffmpeg 完整路径
```
