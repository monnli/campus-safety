# 黔视护苗 · 校园安全智能监控系统
## 用户手册

---

## 一、系统启动

### 1.1 配置环境

把 `backend/.env.example` 复制为 `.env`，填入配置：

```
DASHSCOPE_API_KEY=your-dashscope-api-key   # 阿里云百炼 API Key
YOLO_MODEL_PATH=best.pt                     # 微调后的 YOLO 模型文件名
FFMPEG_PATH=C:\Users\xxx\ffmpeg\bin\ffmpeg.exe  # ffmpeg 完整路径
```

> 即使 API Key 未配置或网络不可用，系统会自动切换到 Mock 模式，保证演示正常进行。

### 1.2 启动后端
```bash
cd campus-safety/backend
pip install -r requirements.txt
python app.py
```

### 1.3 启动前端
```bash
cd campus-safety/frontend
npm install
npm run dev
```

访问 http://localhost:3000，使用以下账号登录：

| 账号 | 密码 | 角色 | 权限 |
|------|------|------|------|
| admin | admin123 | 校长 | 全部功能 |
| safety | safety123 | 安全主任 | 除系统设置外全部 |
| guard | guard123 | 保安 | 监控、事件、历史录像、巡逻 |

## 二、功能模块说明

### 2.1 实时监控

**接入视频源**

点击未接入摄像头格子中的"接入视频"按钮：
- 上传视频文件：拖拽或点击上传本地视频，系统循环播放模拟实时流
- 视频流地址：输入 RTSP 地址、本地视频路径，或输入 `0` 调用电脑摄像头

**检测框显示**

视频接入后，AI 自动开始检测，检测到目标时画面上会实时显示彩色检测框和中文标签（如"打架/暴力 0.96"）。

**摄像头管理**

- 顶部"添加摄像头"按钮：动态添加新摄像头通道
- 每个摄像头右上角：暂停（黄色）、停止（橙色）、删除（红色）
- 超过6路自动分页显示
- 右上角"全屏"按钮：隐藏侧边栏，视频网格铺满屏幕（ESC退出）

---

### 2.2 事件记录

**查看事件列表**

所有经 AI 确认的告警事件按时间倒序排列，支持按行为类型和时间范围筛选。

**事件详情**

点击"详情"查看：
- 事件基本信息（时间、地点、行为类型、置信度）
- Qwen-VL 多帧分析结果（对事件视频片段的时序描述）
- AI 预警报告（事件概述、风险等级、处置建议）
- 存档视频（可在线播放或下载）

**事件处置**

- 标记处理：事件处置完毕后点击，状态更新为"已处理"
- 标记误报：AI 判断有误时点击，状态更新为"误报"

**生成日报**

点击"生成今日日报"，AI 自动汇总当日事件生成安全日报。

---

### 2.3 历史录像

查看持续录像功能录制的完整历史视频。

**筛选录像**：按摄像头和日期筛选，找到需要查看的时间段。

**播放录像**：点击"播放"在弹窗中在线观看，支持进度拖拽。点击"下载"保存到本地。

> 需要先在"系统设置 → 持续录像"中启用录像功能并配置摄像头，才会有录像记录。

---

### 2.4 数据统计

- 事件趋势折线图：近7/14/30天每日事件数量变化，支持多行为类型对比
- 行为类型分布饼图：各类危险行为占比
- 各区域事件分布柱状图：哪个区域事件最多
- 事件时段分布柱状图：哪个时间段最危险
- 校园风险热力图：基于摄像头位置的事件密度可视化

支持时间范围筛选，数据每15秒自动刷新。

---

### 2.5 智能查询

与 AI 安全助手进行自然语言对话：

- "今天发生了哪些事件？"
- "操场今天有几次异常？"
- "最近有入侵事件吗？"
- "哪个区域最危险？"
- "给我一份安全管理建议"

---

### 2.6 巡逻任务

**生成今日任务**

点击"AI 生成今日任务"，系统根据风险分析结果自动规划今日巡逻时间和区域。

**签到确认**

保安人员完成巡逻后点击"签到完成"，管理人员可实时查看完成情况和完成率进度条。

---

### 2.7 无人机规划

**操作流程：**

1. 在搜索框输入学校名称，点击"搜索"按钮自动定位到地图
2. 切换"标记学校"模式，点击地图确认学校中心位置
3. 切换"标记建筑"模式，点击教学楼等建筑物位置，填写名称和高度
4. 切换"画回家路线"模式，沿学生回家路径点击，点击"完成路线"结束，可绘制多条路线
5. 配置飞行参数（基础高度、安全余量、飞行速度、航点间距）
6. 点击"自动生成航点"，系统根据建筑物高度自动计算每个航点的飞行高度
7. 查看右侧飞行信息（总距离、预计时间），确认在电池续航范围内
8. 点击"导出KML航线"，下载航线文件导入大疆 DJI Pilot 2 执行

**地图说明：**
- 蓝色标记：学校中心（起飞/降落点）
- 红色标记：建筑物位置
- 彩色路线：学生回家路线（不同颜色区分）
- 半透明缓冲区：路线两侧的巡逻覆盖范围
- 红色虚线：无人机巡逻航线

---

### 2.8 安全大屏

点击侧边栏"安全大屏"，页面自动进入全屏模式，展示：
- 左栏：校园安全指数、高危区域TOP3、AI风险预测、高危时段分布
- 中栏：实时监控画面（最多4路，自适应布局）、今日事件概览
- 右栏：最新告警（按行为类型区分颜色）、AI防控建议、行为类型统计

数据每30秒自动刷新。点击"退出全屏"或按 ESC 返回。

---

### 2.9 系统设置（仅校长可见）

**检测参数**（修改后立即生效，无需重启）：
- YOLO 置信度阈值（0.1~1.0）
- 检测帧间隔（每N帧检测一次）
- 触发冷却时间（同一摄像头同一行为的最小触发间隔）
- 事件前缓冲时长（触发前保留的视频时长）
- 事件后录制时长（触发后继续录制的时长）
- 检测行为类别（可勾选启用的检测类别）

**通知设置**：管理员列表、短信/警报/广播开关

**语音广播**：
- 广播重复次数（1~10次）
- 各行为类型的自定义广播内容（留空则由AI自动生成）
- 测试广播按钮（可试听效果）

**模型配置**：查看当前模型信息、测试 API 连接状态

**持续录像**：
- 启用开关
- 录像时间段（00:00~23:59 任意时间）
- 分段时长（建议30分钟）
- 指定录像摄像头（建议只选关键位置）
- 录像留存天数（到期自动清理）

**无人机巡逻**：查看巡逻时间段配置、边缘计算模式说明、接入方式说明

**部署与成本**：服务器配置要求、API成本估算、隐私保护声明

## 三、预警响应流程

当系统检测到危险行为时，以下操作自动同步执行：

1. **监控室警报**：页面弹出红色全屏警报弹窗 + 警报音效
2. **语音广播**：AI 生成播报内容，通过浏览器语音合成自动朗读（无需配置，可重复播放）
3. **短信通知**：向管理人员发送短信（Demo 阶段为模拟）
4. **事件记录**：自动保存事件信息和视频片段

收到警报后：
1. 点击"确认知晓"关闭弹窗
2. 前往事件记录页面查看详情和存档视频
3. 派遣相关人员前往现场处置
4. 处置完毕后标记"已处理"；若为误报则标记"误报"

## 四、常见问题

**Q：视频画面不显示？**
A：检查视频文件路径是否正确，或 RTSP 地址是否可访问。确认后端服务已正常启动。

**Q：检测不到目标？**
A：确认 YOLO 模型文件已放在 `backend/` 目录下，且 `.env` 中 `YOLO_MODEL_PATH` 配置正确。可在系统设置中降低置信度阈值。

**Q：Qwen 分析结果显示"分析异常"？**
A：检查 `DASHSCOPE_API_KEY` 是否正确，或网络是否可访问阿里云。系统会自动切换 Mock 模式，不影响演示。

**Q：广播没有声音？**
A：浏览器安全策略要求用户交互后才能播放音频。首次使用时点击页面任意位置后，广播会正常播放。可在系统设置中测试广播效果。

**Q：存档视频播放失败？**
A：确认 ffmpeg 已安装且路径配置正确（`.env` 中 `FFMPEG_PATH`）。新触发的事件会自动转码为 H.264 格式。

**Q：历史录像为空？**
A：需要先在"系统设置 → 持续录像"中启用录像功能，选择摄像头并保存配置，在录像时间段内接入视频后才会开始录制。

**Q：修改检测参数后不生效？**
A：检测参数修改后点击"保存设置"即立即生效，无需重启后端。

**Q：无人机规划地图搜索不到学校？**
A：尝试输入更完整的学校名称（如"贵阳信息科技学院"），或切换"标记学校"模式手动在地图上点击标记。

**Q：重启后摄像头接入状态丢失？**
A：系统会自动恢复上次的摄像头接入状态（视频文件路径需仍然存在）。如果视频文件已移动或删除，需要重新接入。
